# Contributing

Joch AI is pre-MVP. Contributions should favor the clean end-state over backward compatibility.

## Contribution focus

Good early contributions:

```text
MVP resource schemas
CLI behavior
gateway policy checks
MCP scanning
tool side-effect classification
approval workflow
ABOM generation
framework adapters
examples
docs that clarify the MVP wedge
```

Avoid:

```text
new agent loops
custom planning frameworks
advanced RAG platform work
marketplace features
large SaaS-only surfaces
```

## Documentation rule

Docs should keep repeating the same strategic boundary:

> Joch AI does not replace agent frameworks. It governs the agents teams already build.
